# Century 21 Smart CRM - 프로젝트 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**Century 21 Smart CRM** - 부동산 중개업 고객관계관리 시스템

### 1.2 프로젝트 목적
부동산 중개사(공인중개사)의 고객 관리 업무를 체계화하고, 영업 활동을 정량화하여 효율적인 고객 관계 관리를 지원하는 웹 기반 CRM 서비스

### 1.3 타겟 사용자
- **Primary**: Century 21 소속 공인중개사 (에이전트)
- **Secondary**: 지점 관리자, 마스터 관리자

### 1.4 프로젝트 범위 (MVP)
| 구분 | 범위 |
|------|------|
| 플랫폼 | 웹 (PC + 모바일 반응형) |
| 백엔드 | 미구현 (Mock Data 활용) |
| 인증 | 미구현 (UI만 존재) |
| 데이터 저장 | 브라우저 메모리 (새로고침 시 초기화) |

## 3. 사용자 역할 및 권한

### 3.1 역할 정의
| 역할 | 설명 | 접근 가능 메뉴 |
|------|------|---------------|
| 에이전트 | 일반 공인중개사 | 대시보드, 고객DB, 일정, 통계, AI비서, 운영관리 |
| 마스터 관리자 | 지점/전체 관리자 | 전체 메뉴 + 마스터관리, 중개사관리 |

### 3.2 MVP 현황
- 역할 구분 UI는 존재하나 **실제 권한 제어 미구현**
- 모든 메뉴 접근 가능 상태

---

## 4. 핵심 가치 제안

1. **고객 관계의 체계적 관리**: 고객 정보를 한 곳에서 관리하고, Contact 주기를 자동 알림
2. **영업 활동의 정량화**: 목표 설정 및 달성률 추적을 통한 성과 관리
3. **효율적인 마케팅**: 고객 그룹화를 통한 타겟 마케팅 지원
4. **모바일 친화적**: PC와 모바일에서 동일한 사용자 경험 제공
