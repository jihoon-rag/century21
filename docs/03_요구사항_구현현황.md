### 2.2 구현 상세

#### 1-1. 고객등록
```
경로: 고객 DB → 등록 버튼 → 신규 고객 등록 화면
필수 입력: 고객명, 연락처
선택 입력: 이메일, 주소, 메모, 프로필 사진, 그룹
```

#### 1-4. 대량업로드기능
```
경로: 고객 DB → 업로드 버튼 → Excel 업로드 모달
지원 형식: CSV (UTF-8)
필수 컬럼: 고객명, 연락처
선택 컬럼: 이메일, 주소, 메모
템플릿 다운로드 제공
```

#### 1-6. 파일첨부기능
```
경로: 고객 DB → 고객 선택 → 상세 페이지 → 첨부 파일 섹션
저장 방식: Base64 인코딩 (브라우저 메모리)
파일 정보: 파일명, 타입, 크기, 업로드일
```

## 3. Phase 2 - 고객DB 관리

### 3.1 요구사항 및 구현 현황

| # | 요구사항 | 구현 상태 | 구현 위치 | 비고 |
|---|---------|:--------:|----------|------|
| 2-1 | 고객분류(그룹화) | ✅ 완료 | `components/GroupManageModal.tsx` | 그룹 CRUD, 색상 지정 |
| 2-2 | Push 알림기능 | ⚠️ UI만 | `components/NotificationPanel.tsx` | 실제 푸시 미연동 |
| 2-3 | Calendar 연동 | ✅ 완료 | `views/ScheduleView.tsx` | 일정 관리, 날짜별 이벤트 |
| 2-4 | Data추출기능(Excel) | ✅ 완료 | `views/CustomerDB.tsx` | CSV 다운로드 |

### 3.2 구현 상세

#### 2-1. 고객분류(그룹화)
```
그룹 관리 경로: 고객 DB → 그룹 버튼 → 그룹 관리 모달
- 그룹 생성: 이름, 색상, 설명
- 그룹 수정/삭제
- 그룹별 고객 수 표시

고객 그룹 할당 경로: 고객 상세 → 그룹 설정 섹션
- 복수 그룹 선택 가능
- 그룹 태그로 표시

필터링 경로: 고객 DB → 그룹 필터
- 선택한 그룹에 속한 고객만 표시
```

#### 2-4. Data추출기능
```
경로: 고객 DB → 다운로드 버튼
파일명: 고객목록_YYYY-MM-DD.csv
포함 컬럼: 고객명, 연락처, 이메일, 주소, 최근연락, 등록일, 상태, 메모
인코딩: UTF-8 BOM (한글 깨짐 방지)
필터 적용: 현재 필터링된 고객만 추출
```

### 4.2 구현 상세

#### 3-1. Data의 시각화
```
위치: 통계 페이지

날짜 이동: 이전/다음 버튼
데이터: 신규 등록 수, Contact 기록 수

```

```
목표 유형: 월간, 분기별
설정 항목: 신규 등록 목표(명), Contact 목표(건)
달성률 표시: 프로그레스 바 + 퍼센트
100% 달성 시: 녹색 표시
```
