# Century 21 Smart CRM - FAQ 및 예상 질문

## 1. 알림/푸시 관련

### Q1. 푸시 알림은 누가 보내는 건가요? 마스터가 보내나요?

**A.** 현재 MVP에서 푸시 알림은 **시뮬레이션**으로만 동작합니다.

**실서비스 구현 시 알림 종류:**

| 알림 유형 | 발송 주체 | 대상 | 내용 |
|----------|----------|------|------|
| Contact 기한 알림 | **시스템 자동** | 에이전트 | "홍길동 고객 연락 기한이 3일 지났습니다" |
| 공지사항 | **마스터** | 전체 에이전트 | 회사 공지, 정책 변경 등 |
| 일정 알림 | **시스템 자동** | 에이전트 | "오후 2시 미팅이 있습니다" |
| 신규 고객 | **시스템 자동** | 에이전트/마스터 | "신규 고객이 등록되었습니다" |

**구현 위치:**
- UI: `components/NotificationPanel.tsx`
- 데이터: `context/AppContext.tsx` → `notifications`

**백엔드 연동 시 필요 작업:**
- FCM(Firebase Cloud Messaging) 연동
- 알림 스케줄러 구축 (Contact 기한 체크)
- 알림 히스토리 저장

---

### Q2. 사용자(공인중개사)가 고객에게 문자를 보내면 비용은 어떻게 되나요?

**A.** 현재 MVP에서는 **실제 문자가 발송되지 않습니다** (UI 시뮬레이션).

**실서비스 구현 시 비용 구조:**

| 항목 | 비용 | 비고 |
|------|------|------|
| SMS (단문) | 10-15원/건 | 90byte 이하 |
| LMS (장문) | 30-40원/건 | 2,000byte 이하 |
| MMS (멀티미디어) | 100-200원/건 | 이미지 포함 |

**비용 부담 방식 (선택지):**
1. **회사 부담**: 월정액 + 사용량 비용을 회사가 부담
2. **에이전트 개인 부담**: 에이전트별 충전금 또는 건당 과금
3. **하이브리드**: 기본 제공량 초과 시 개인 부담

**추천 SMS Gateway:**
- 솔라피 (solapi.com)
- 알리고 (aligo.in)
- NHN Cloud Notification

**구현 위치:**
- UI: `views/OperationalManagement.tsx`

---

### Q3. 카카오톡 알림은 안 되나요?

**A.** 현재 MVP에는 **미구현**이며, 추가 개발이 필요합니다.

**카카오 알림톡 구현 시:**

| 항목 | 내용 |
|------|------|
| 서비스 | 카카오 비즈니스 채널 + 알림톡 API |
| 비용 | 8-15원/건 (SMS보다 저렴) |
| 장점 | 수신율 높음, 브랜드 신뢰도 |
| 단점 | 템플릿 사전 승인 필요 (1-3일 소요) |

**필요 절차:**
1. 카카오 비즈니스 채널 개설
2. 알림톡 발신 프로필 등록
3. 메시지 템플릿 등록 및 승인
4. API 연동 개발

**예상 공수:** 4-5일

**적합한 사용 사례:**
- Contact 기한 알림
- 예약/일정 알림
- 계약 진행 상황 안내

## 2. 데이터 관련

### Q4. 데이터는 어디에 저장되나요?

**A.** 현재 MVP는 **브라우저 메모리**에 저장됩니다.

| 현재 상태 | 동작 |
|----------|------|
| 새로고침 | 모든 데이터 초기화 (Mock Data로 리셋) |
| 브라우저 종료 | 데이터 소멸 |
| 다른 기기 | 데이터 공유 안됨 |

**실서비스 구현 시:**
- 클라우드 데이터베이스 (Supabase, Firebase 등)
- 모든 기기에서 동일 데이터 접근
- 영구 저장 및 백업

---

### Q5. Excel 업로드 시 기존 고객과 중복 체크되나요?

**A.** 현재 MVP에서는 **중복 체크 없이 모두 신규 등록**됩니다.

**실서비스 권장 로직:**
1. 연락처(전화번호) 기준 중복 체크
2. 중복 시 옵션 제공:
   - 건너뛰기
   - 기존 데이터 업데이트
   - 별도 표시 후 수동 처리

**구현 위치:** `components/ExcelUploadModal.tsx`

## 3. 권한/보안 관련

### Q6. 에이전트 간 고객 정보가 공유되나요?

**A.** 현재 MVP에서는 **권한 구분 없이 모든 데이터 접근 가능**합니다.

**실서비스 권장 구조:**

| 역할 | 접근 범위 |
|------|----------|
| 에이전트 | 본인이 등록한 고객만 |
| 팀장 | 팀원들의 고객 열람 가능 |
| 마스터 | 전체 고객 접근 |

**구현 시 고려사항:**
- Row Level Security (Supabase 지원)
- 고객 테이블에 `agent_id` 필드 추가
- API에서 인증된 사용자의 데이터만 반환

## 4. 기능 관련

### Q8. AI 비서는 실제로 동작하나요?

**A.** 현재 MVP에서는 **키워드 매칭 기반 응답**으로 동작합니다.

**현재 동작 방식:**
```javascript
// 예: "요약" 키워드 입력 시
if (message.includes('요약')) {
  return "이번 주 활동을 요약해드릴게요...";
}
```

**실제 AI 연동 시:**
- Gemini API 또는 OpenAI API 연결
- 고객 데이터 기반 맥락 제공
- 더 자연스럽고 개인화된 응답

**구현 위치:** 
- UI: `views/AISecretary.tsx`
- 서비스: `services/geminiService.ts` (현재 미사용)

---

### Q9. 연락 기한(Contact 도래)은 어떻게 계산되나요?

**A.** 고객별로 설정된 **Contact 주기**를 기준으로 계산됩니다.

**계산 로직:**
```
Contact 도래일 = 마지막 연락일 + Contact 주기
남은 일수 = Contact 도래일 - 오늘
```

**표시 방식:**
- `+3일`: 3일 후 연락 필요
- `Today`: 오늘 연락 필요
- `-1일`: 1일 지남 (빨간색)

**Contact 주기 설정:**
- 고객 상세 페이지에서 7/14/30/60일 중 선택
- 기본값: 14일

**구현 위치:** 
- 설정: `views/CustomerDetail.tsx`
- 표시: `views/CustomerDB.tsx`

---

### Q10. 통계 데이터는 실시간인가요?

**A.** 현재 MVP에서는 **하드코딩된 샘플 데이터**입니다.

**현재 상태:**
- 요약 카드: 실제 고객 데이터 기반 계산 ✅
- 차트 데이터: 샘플 데이터 (고정값)

**실서비스 구현 시:**
- 모든 통계 실시간 집계
- 기간별 데이터 DB 조회
- 캐싱으로 성능 최적화

**구현 위치:** `views/StatisticsView.tsx`

## 6. 기술적 질문

### Q13. 오프라인에서도 사용 가능한가요?

**A.** 현재 MVP에서는 **오프라인 지원 없음** (인터넷 필수).

**PWA(Progressive Web App) 구현 시:**
- 오프라인 캐싱 가능
- 홈 화면에 앱 설치
- 추가 개발 필요 (2-3일)

---

### Q14. 모바일 앱은 없나요?

**A.** 현재는 **웹 전용**이며, 모바일 브라우저 최적화되어 있습니다.

**앱 개발 옵션:**

| 방식 | 장점 | 단점 | 예상 공수 |
|------|------|------|----------|
| PWA | 빠른 개발, 앱스토어 불필요 | 일부 기능 제한 | 1-2주 |
| React Native | 네이티브 성능 | 별도 코드베이스 | 8-12주 |
| Flutter | 높은 성능 | 학습 곡선 | 8-12주 |

## 8. 향후 기능 질문

**A.** 우선순위별 추천 기능:

1. 사용자 인증/권한 관리
2. 실제 데이터 저장
3. 알림 서비스 연동

**중간:**
1. SMS/알림톡 연동
2. 매물 연동 (부동산 API)

1. AI 고객 분석
2. 자동 마케팅 캠페인
3. 리포트 자동 생성/이메일 발송

**A.** 현재 MVP는 독립 시스템이며, **API 개발 시 연동 가능**합니다.

- 부동산 매물 플랫폼 (직방, 다방 등)
